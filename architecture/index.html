Bootstraping

  • loading and starting the application,
  • provides control over the moment the application is launched,
  • allows to perform actions after the application starts,
  • allows to react when an error occurs during application start.

Bootstraping

main.ts

  if (environment.production) {
    enableProdMode();
  }
  
  platformBrowserDynamic()
    .bootstrapModule(AppModule)
    .then(()=> console.log('App started!'))
    .catch((err) => console.log(err));

Components

  • control the view,
  • without business logic,
  • defines metadata: selector, styles, body,
  • communication:
    • parent -> child – passing data via properties (binding),
    • child -> parent – emitting Angulars events.

Components

Custom DOM elements

  import { Component } from '@angular/core';

  @Component({
    selector: 'jstk-root',
    templateUrl: './app.component.html',
    styleUrls: ['./app.component.scss']
  })
  export class AppComponent {
    title = 'jstk-app';
  }

Pipes

Data transformations

  • allow to transform data, mainly in html,
  • chaining with | (linux style),
  • support for memoization (pure pipes).

<p>Today is {{today | date}}</p>


<p>The date is {{today | date: 'fullDate'}}</p>


<p>The time is {{today | date:'shgrtTime'}}</p>

Directives

DOM transformations

  • allow to add superpowers to the DOM elements,
  • manipulating DOM tree,
  • build-in directives:
    • *ngIf,
    • *ngFor,
    • ngSwitch & *ngSwitchCase,
    • ngModel,
    • ngClass.

Services

  • reusability and modularity of existing code,
  • contain main application logic,
  • obtained from dependency injection,
  • must be provided in root or module.
  import { Injectable } from '@angular/core';
  import { StarterService } from './starter.service';
  @Injectable({
    prowidedIn: 'root'
  })
  export class AppServiceService {
    constructor(private starterService: StarterService) {}
  }

Modules

  • build-in module system – NgModules,
  • must be at last one,
  • agregate domain, flow or similar functionality,
  • modules can be nested.

Modules

  import { BrowserModule } from '@angular/platform-browser';
  import { NgModule } from '@angular/core';

  import { AppRoutingModule } from './app-routing.module';
  import { AppComponent } from './app.component';
  
  @NgModule({
    declarations: [
      AppComponent
    ],  
    imports: [
      BrowserModule,
      AppRoutingModule
    ],
    providers: [],
    bootstrap: [AppComponent]
  })
  export class AppModule { }

Modules

  • imports – external modules,
  • exports – exported elements,
  • declarations - components and pipes (for template proposes),
  • providers – injectables (services, guards, pipes),
  • bootstrap – starting point for application.
